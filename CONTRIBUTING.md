# Contributing to Open LLM Orchestrator

We welcome contributors interested in distributed systems, workflow orchestration, model serving, and scalable AI infrastructure.

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Open a Pull Request

## Coding Principles

- Keep modules loosely coupled
- Maintain language-neutral interfaces
- Prefer simplicity over unnecessary abstraction
- Ensure observability is built-in

## Areas Open for Contribution

- Model SDK (gRPC contracts)
- Temporal workflows
- Model routing strategies
- RAG pipeline improvements
- Helm charts and deployment tooling

Please open an issue before large architectural changes.
